# Purchase API — Дипломный проект: автоматизация закупок

REST API для автоматизации закупок и оформления заказов в розничной сети. Проект реализован как MVP на Django + Django REST Framework.

## Цель проекта

Автоматизировать процесс закупок: быстрый импорт прайсов, формирование заказов из корзины, автоматический расчёт стоимости и отправка уведомлений. Проект демонстрирует работу с Django ORM, API-интеграцией и управлением зависимостями.

## Функционал (базовая часть выполнена)

✅ **Создание заказа через POST-запрос.**  
✅ **Привязка товара по ID** — защита от опечаток в названиях и гарантия корректного расчёта суммы.  
✅ **Автоматический расчёт итоговой стоимости** на основе актуальной цены из БД.  
✅ **Отправка email-уведомлений** пользователю (в режиме разработки письма выводятся в консоль сервера).  
✅ **Импорт товаров из CSV** через команду управления `import_products`.  

## Архитектурные решения (осознанные ограничения)

- **Без Celery:** импорт товаров выполняется синхронно через CLI-команду. Для MVP и небольших объёмов данных это оптимальное решение: оно проще в поддержке и полностью закрывает задачу.  
- **Без троттлинга:** защита от высоконагруженных сценариев не требуется на этапе MVP.  
- **Без интеграций с соцсетями:** не входят в целевую функциональность проекта «автоматизация закупок».  

## Технологии

- Python 3.10+
- Django
- Django REST Framework
- SQLite (для разработки)
- drf-spectacular (Swagger)
- JWT-авторизация
